# 여운 IA (Information Architecture)

## 문서 목적

이 문서는 **여운** 앱의 정보 구조(IA)를 정의한다.  
목표는 다음과 같다.

- 사용자가 어떤 화면에서 어떤 행동을 하는지 명확히 정의한다.
- 디자이너와 개발자가 동일한 화면 구조를 기준으로 작업할 수 있게 한다.
- MVP 범위와 이후 확장 범위를 분리해 우선순위를 정리한다.

---

# 1. 제품 구조 한눈에 보기

## 1-1. 핵심 제품 구조

여운은 다음 5개의 핵심 축으로 구성된다.

1. **홈**: 추천/최신/팔로우 중심 피드 소비
2. **산책**: 새로운 게시물/유저/무드를 발견하는 탐색
3. **업로드**: 일반 게시물 또는 찰나 업로드
4. **편지함**: 1:1 대화 및 메시지 요청 관리
5. **무드카드**: 프로필, 게시물, 저장, 설정 진입

## 1-2. 콘텐츠 구조

여운에는 게시물이 2종류 있다.

### A. 일반 게시물

- 프로필 그리드에 남음
- 저장 가능
- 댓글 가능
- 여운 반응 가능

### B. 찰나

- 12시간 또는 24시간 뒤 자동 만료
- 홈 상단에서 소비
- 프로필 메인 그리드에 영구 저장되지 않음
- 여운 반응 가능
- 짧은 문구 중심

# 2. 전역 내비게이션 구조

## 2-1. 하단 탭바

여운의 기본 하단 탭바는 5개로 구성한다.

1. **홈**
2. **산책**
3. **업로드**
4. **편지함**
5. **무드카드**

## 2-2. 전역 공통 진입 요소

화면에 따라 다음 공통 진입 요소를 둔다.

- 알림함 진입
- 검색 진입
- 설정 진입
- 게시물 상세 진입
- 유저 프로필 진입
- 신고/차단 액션

# 4. 화면 정의

# 4-1. 스플래시

## 목적

앱 진입 시 브랜드 인지와 초기 로딩 상태를 제공한다.

## 핵심 요소

- 여운 로고
- 짧은 브랜드 카피
- 자동 로그인 체크

## 이동

- 로그인 상태 유지 시 홈으로 이동
- 미로그인 시 온보딩 또는 로그인으로 이동

---

# 4-2. 온보딩

## 목적

초기 사용자가 서비스 성격을 이해하고 관심 무드를 선택하도록 돕는다.

## 하위 화면

### A. 서비스 소개

- 여운이 무엇인지 설명
- 좋아요가 아닌 여운 반응 구조 안내

### B. 관심 무드 선택

- 몽환, 청량, 빈티지, 힙, 새벽, 필름 등 복수 선택
- 추천 피드 초기화에 사용

### C. 알림 권한 요청

- 반응, 댓글, 편지, 찰나 알림 안내

## 이동

- 온보딩 완료 → 회원가입 또는 로그인

---

# 4-3. 인증

## 로그인

### 목적

기존 사용자의 앱 재진입

### 기능

- 이메일 로그인
- 소셜 로그인
- 비밀번호 재설정 진입

### 이동

- 성공 시 홈 이동

## 회원가입

### 목적

신규 계정 생성

### 기능

- 이메일/비밀번호 입력
- 닉네임 입력
- 프로필 사진 설정
- 자기소개 입력(선택)

### 이동

- 가입 완료 시 홈 또는 프로필 설정 완료 화면

---

# 4-4. 홈

## 목적

사용자가 여운을 남기고 사진을 소비하는 메인 화면

## 핵심 구조

1. 상단 헤더
2. 찰나 영역
3. 피드 탭
4. 게시물 피드
5. 플로팅 업로드 버튼

## 세부 구성

### A. 상단 헤더

- 서비스명 여운
- 알림함 버튼
- 검색 버튼

### B. 찰나 영역

- 내 찰나 올리기 카드
- 타 유저 찰나 가로 스크롤
- 남은 시간 표시

### C. 피드 탭

- 추천
- 최신
- 팔로우

### D. 게시물 카드

- 작성자 썸네일
- 닉네임
- 무드 태그
- 업로드 시간
- 사진
- 대표 반응 + 평균 감도
- 참여 인원
- 설명 1줄
- 여운 액션
- 댓글 진입
- 저장

### E. 플로팅 업로드 버튼

- 일반 게시물/찰나 업로드 진입

## 주요 액션

- 여운 반응 남기기
- 댓글 보기
- 저장
- 프로필 보기
- 찰나 보기

---

# 4-5. 게시물 상세

## 목적

일반 게시물을 깊게 소비하고 상호작용한다.

## 핵심 요소

- 사진 전체
- 작성자 정보
- 설명 전체
- 무드 태그
- 대표 반응 + 평균 감도
- 참여 인원
- 여운 5단 반응 버튼
- 댓글 목록
- 댓글 작성
- 저장
- 신고

## 주요 액션

- 여운 반응 변경
- 댓글 작성/삭제
- 저장/저장 취소
- 신고
- 작성자 프로필 이동

---

# 4-6. 찰나 상세

## 목적

시간 제한이 있는 휘발성 게시물을 소비한다.

## 핵심 요소

- 사진
- 짧은 문구
- 무드 태그
- 남은 시간
- 여운 5단 반응
- 짧은 답장 또는 반응
- 신고

## 제약

- 만료 후 접근 불가
- 저장 기능은 제거하거나 제한

---

# 4-7. 산책

## 목적

새로운 게시물, 태그, 사람을 발견하는 탐색 허브

## 핵심 구조

1. 검색창
2. 인기 태그
3. 추천 유저
4. 격자 피드
5. 필터 탭

## 세부 구성

### A. 검색창

- 닉네임, 태그, 설명 검색

### B. 인기 태그 영역

- 실시간/주간 인기 태그

### C. 추천 유저

- 내 관심 무드 기반 추천

### D. 격자 피드

- 게시물 썸네일 모자이크 그리드
- 대표 반응과 평균 감도 최소 노출 가능

### E. 필터 탭

- 최신순
- 인기순
- 태그별
- 사용자별

## 주요 액션

- 게시물 상세 진입
- 태그 탐색
- 유저 프로필 진입
- 팔로우

---

# 4-8. 업로드

## 목적

일반 게시물 또는 찰나를 생성한다.

## 업로드 진입 구조

### A. 업로드 유형 선택

- 일반 게시물
- 찰나

### B. 사진 선택

- 앨범에서 선택
- 카메라 촬영 가능 여부는 추후 결정

### C. 정보 입력

- 설명 입력
- 무드 태그 선택
- 공개 범위 선택
- 위치 입력(선택)

### D. 찰나 추가 입력

- 12시간 / 24시간 선택

### E. 업로드 완료

- 홈 또는 상세 화면으로 이동

## 제약

- 일반 게시물은 1~4장
- 찰나는 1장 권장

---

# 4-9. 편지함

## 목적

사용자 간 1:1 대화를 관리한다.

## 핵심 구조

1. 대화 목록
2. 메시지 요청함
3. 채팅방

## 세부 구성

### A. 대화 목록

- 최근 대화 순 정렬
- 읽지 않음 표시

### B. 메시지 요청함

- 새 대화 요청 수락/거절

### C. 채팅방

- 텍스트 메시지
- 상대 프로필 진입
- 차단/신고

## 정책

- 기본은 팔로우한 사용자만 대화 가능 또는 메시지 요청 후 수락 방식

---

# 4-10. 알림함

## 목적

반응, 댓글, 팔로우, 찰나, 편지 관련 알림을 모아서 보여준다.

## 알림 카테고리

- 여운 알림
- 댓글 알림
- 팔로우 알림
- 찰나 알림
- 편지 알림

## 알림 클릭 시 이동

- 게시물 상세
- 찰나 상세
- 채팅방
- 프로필

---

# 4-11. 무드카드(프로필)

## 목적

사용자 정체성과 게시물 이력을 보여준다.

## 핵심 요소

- 프로필 사진
- 닉네임
- 자기소개
- 대표 무드 태그
- 평균 감도
- 받은 여운 수
- 팔로워 / 팔로잉
- 게시물 그리드
- 현재 찰나 표시
- 보관함 진입
- 프로필 편집

## 주요 액션

- 프로필 편집
- 팔로워/팔로잉 확인
- 저장한 게시물 보기
- 설정 이동

---

# 4-12. 보관함

## 목적

저장한 게시물을 다시 소비한다.

## 핵심 요소

- 저장 게시물 목록
- 태그 필터
- 정렬 옵션

## 이동

- 게시물 상세
- 작성자 프로필

---

# 4-13. 설정

## 목적

계정, 알림, 공개 범위, 안전 관련 설정을 관리한다.

## 하위 항목

- 계정 설정
- 알림 설정
- 공개 범위 설정
- 메시지 허용 범위 설정
- 차단 목록
- 로그아웃
- 탈퇴

# 5. 사용자 플로우

# 5-1. 신규 사용자 플로우

```text
스플래시 → 온보딩 → 관심 무드 선택 → 회원가입 → 홈 진입
```

# 5-2. 일반 게시물 업로드 플로우

```text
홈 또는 탭바 업로드 → 업로드 유형 선택(일반 게시물) → 사진 선택 → 설명/태그 입력 → 업로드 완료 → 게시물 상세 또는 홈
```

# 5-3. 찰나 업로드 플로우

```text
홈 찰나 영역 또는 탭바 업로드 → 업로드 유형 선택(찰나) → 사진 선택 → 짧은 문구/태그 입력 → 12h 또는 24h 선택 → 업로드 완료 → 홈 상단 찰나 영역 노출
```

# 5-4. 여운 반응 플로우

```text
홈 피드 또는 상세 → 발바닥 단계 선택 → 반응 적용 → 평균 감도/대표 반응 갱신
```

# 5-5. 탐색 플로우

```text
산책 진입 → 태그/검색/격자 피드 탐색 → 게시물 상세 → 여운/댓글/팔로우
```

# 5-6. 관계 형성 플로우

```text
게시물 상세 또는 프로필 → 팔로우 → 편지함 요청 또는 대화 시작 → 채팅방
```

# 8. 네이밍 가이드

## 사용자 노출 용어

- 여운 = 발바닥 반응 시스템
- 찰나 = 사라지는 휘발성 게시물
- 산책 = 탐색 탭
- 편지함 = DM 탭
- 무드카드 = 프로필 탭
- 보관함 = 저장 탭/화면
- 평균 감도 = 평균 점수 정보

## 피해야 할 용어

- 좋아요
- 별점
- 스토리
- 탐색(가능하면 내부 용어로만 사용)
- DM(유저 노출 텍스트로는 최소화)

# 10. Codex 작업 지시용 요약

이 IA를 기준으로 구현할 때 아래 원칙을 따른다.

1. 모든 사용자 흐름은 **모바일 우선**으로 설계한다.
2. 홈과 산책의 역할을 혼동하지 않는다.
   - 홈 = 관계 중심 소비
   - 산책 = 발견 중심 소비
3. 찰나는 일반 게시물과 구분된 별도 콘텐츠 타입으로 다룬다.
4. 여운 반응은 좋아요나 별점처럼 보이지 않도록 UI와 용어를 유지한다.
5. 편지함은 반드시 안전장치(요청/수락/차단/신고)를 포함한다.
6. 알림함은 게시물/찰나/편지/팔로우 상호작용을 다시 앱으로 유도하는 핵심 허브로 본다.
