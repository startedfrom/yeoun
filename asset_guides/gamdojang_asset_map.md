# 여운 에셋 맵

## 1. 기본 원칙

- 모바일 퍼스트 기준으로 잡는다.
- 사진은 장식보다 우선한다.
- 아이콘은 하나의 픽셀 패밀리처럼 보여야 한다.
- 같은 사진을 여러 화면에서 재활용할 수 있도록 원본 비율을 먼저 통일한다.
- 원본 생성은 여유 해상도로 만들고, 앱 안에서는 파생 크롭을 만든다.

## 2. 파일 네이밍 규칙

- 아이콘: `ic-{name}-{size}.svg`
- 브랜드 특화 아이콘: `ic-brand-{name}-{size}.svg`
- 피드 사진: `img-feed-{slug}-{ratio}.jpg`
- 찰나 사진: `img-chalna-{slug}-{ratio}.jpg`
- 프로필 이미지: `img-avatar-{slug}-1x1.jpg`
- 빈 상태 일러스트: `ill-{screen}-empty.png`
- 온보딩/스플래시: `ill-{screen}-hero.png`
- 스티커/배지: `sticker-{name}.png`

예시:

- `ic-home-24.svg`
- `ic-brand-paw-24.svg`
- `img-feed-dreamy-room-4x5.jpg`
- `img-chalna-night-walk-9x16.jpg`
- `ill-mailbox-empty.png`

## 4. 사진 / 일러스트 슬롯 맵

### 4.1 공통 생성 기준

- 일반 게시물 원본: `4:5`, 최소 `1440 x 1800`
- 찰나 원본: `9:16`, 최소 `1080 x 1920`
- 프로필/아바타 원본: `1:1`, 최소 `1024 x 1024`
- 빈 상태 / 온보딩 일러스트: `3:4` 또는 `4:5`, 최소 `1536 x 2048`
- 스티커 / 배지: `1:1`, 최소 `1024 x 1024`, 가능하면 투명 배경

## 5. 화면별 배치 메모

### 스플래시

- 사진보다 브랜드 일러스트가 중심이다.
- 로고 아래쪽 또는 중앙 아래에 카피가 들어가므로, 일러스트 중심 피사체는 정중앙보다 약간 위에 둔다.

### 온보딩

- 상단 절반 가까이에 메인 일러스트가 오고, 하단에는 설명과 CTA가 온다.
- 캐릭터/장식이 화면 가장자리에 머물고, 중앙 텍스트 영역을 침범하지 않게 한다.

### 홈

- 찰나는 상단 가로 스크롤 타일이므로 원본은 세로형으로 생성하되, 실제 목록에서는 3:4로 잘려도 주요 피사체가 살아야 한다.
- 피드 메인 사진은 전체 카드의 감정선을 좌우하므로 4:5 원본을 기준으로 통일하는 것이 가장 안정적이다.

### 산책

- 산책은 정보보다 발견성이 중요하므로, 정사각형 썸네일 크롭에서 이미 분위기가 읽혀야 한다.
- 너무 가장자리 구도는 1:1 크롭에서 잘릴 위험이 크다.

### 무드카드

- 아바타는 배경보다 인물/오브젝트가 명확해야 한다.
- 프로필 그리드는 1:1 크롭으로 반복되므로, 원본 사진마다 중심 소재가 선명해야 한다.

### 편지함

- 아바타는 작은 크기에서도 식별 가능해야 하므로, 얼굴/실루엣/헤어/액세서리 대비가 분명한 이미지를 쓴다.
- 빈 상태 일러스트는 텍스트가 같이 들어가므로 하단 20~25%를 비워둔다.

## 6. 실제 생성 우선순위

### P0

- 탭바/글로벌 아이콘 세트
- 홈 피드용 4:5 사진 8~12장
- 찰나용 9:16 사진 4~6장
- 프로필 아바타 6~8장
- 스플래시 / 온보딩 일러스트 2종

### P1

- 편지함 / 알림함 / 보관함 빈 상태 일러스트
- 업로드 완료 스티커
- 무드 태그 스티커성 장식

### P2

- 시즌성 스탬프
- 이벤트 배지
- 프로모션용 배너 아트

## 7. 구현 메모

- 피드 / 산책 / 무드카드 썸네일은 별도 생성보다 원본에서 파생 크롭하는 편이 일관성 유지에 유리하다.
- 아이콘은 `SVG` 또는 코드 컴포넌트가 우선이다.
- 이미지 생성은 시리즈별로 한 번에 몰아서 하고, 베스트 결과 1장을 레퍼런스로 고정하면 통일감이 좋아진다.
