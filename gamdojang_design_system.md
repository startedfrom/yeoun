# 여운 디자인 시스템 (Design System)

## 문서 목적

이 문서는 **여운** 앱의 디자인 원칙과 시각 언어를 일관되게 유지하기 위한 디자인 시스템 기준서다.  
목표는 다음과 같다.

- 파스텔 픽셀 세계관을 모든 화면에서 일관되게 유지한다.
- 인스타/소개팅앱의 익숙한 사용성을 참고하되, 결과물은 여운만의 서비스처럼 보이게 한다.
- 디자이너, 프론트엔드 개발자, AI 코딩 에이전트(Codex/Cursor 등)가 같은 규칙으로 화면을 만들 수 있게 한다.
- 귀엽지만 유아틱하지 않고, 감성적이지만 싸구려처럼 보이지 않는 톤을 유지한다.

---

# 1. 브랜드 방향

## 1.1 브랜드 키워드

- 파스텔 픽셀
- 고양이 발바닥
- 무드 태그
- 젤리 반응
- 사진보다 감도
- 귀엽지만 과하지 않음
- 수집하고 싶은 분위기
- 부드럽고 몽글한 상호작용

## 1.2 브랜드 인상

여운은 단순한 사진 평가 앱이 아니다.  
사용자가 사진의 외모나 점수를 평가받는 공간이 아니라, **사진의 분위기와 감도를 남기고 수집하는 공간**처럼 보여야 한다.

따라서 디자인은 아래 인상을 전달해야 한다.

- 첫인상은 귀엽고 부드럽다.
- 사용성은 익숙하고 쉽게 이해된다.
- 반응 구조는 공격적이지 않고 말랑하다.
- 화면은 게임처럼 세계관이 있으나, 실제 서비스처럼 정돈되어 있다.

## 1.3 피해야 하는 인상

- 흔한 보라색 그라데이션 스타트업 앱
- 카드 위에 카드가 겹치는 과한 SaaS 레이아웃
- 유아용 앱처럼 과도하게 둥글고 장난감 같은 표현
- 인스타/틴더의 레이아웃과 아이콘을 거의 그대로 베낀 느낌
- 하트, 별점, 불꽃 등 너무 흔한 SNS/매칭앱 시각 언어

# 2. 디자인 원칙

## 2.1 사진이 주인공이어야 한다

여운은 결국 사진 기반 서비스다.  
배경, 장식, 캐릭터가 예뻐도 사진을 가리거나 이기면 안 된다.

원칙:

- 장식보다 사진을 우선한다.
- 사진 영역은 충분히 크게 사용한다.
- 텍스트와 버튼은 사진 감상을 방해하지 않아야 한다.

## 2.2 귀여움은 규칙으로 만들고, 장식으로 해결하지 않는다

여운의 귀여움은 리본과 스티커를 마구 붙여서 만드는 것이 아니다.

귀여움은 아래의 조합에서 나와야 한다.

- 여백이 넉넉한 배치
- 부드러운 컬러 톤
- 픽셀 기반 아이콘 언어
- 짧고 말랑한 마이크로카피
- 일관된 반응 애니메이션

## 2.3 익숙하되, 정체성은 분명하게 한다

홈 피드, 탐색, 프로필, DM은 사용자가 익숙하게 접근할 수 있어야 한다.  
하지만 하트, 스토리, DM, 돋보기 등 흔한 SNS 표현을 그대로 사용하지 않고 여운만의 용어와 형태로 바꿔야 한다.

예:

- 좋아요 → 여운
- 스토리 → 찰나
- DM → 편지함
- 탐색 → 산책
- 프로필 → 무드카드

## 2.4 귀엽지만 읽기 쉬워야 한다

폰트와 아이콘이 귀여워도 정보 전달력이 떨어지면 안 된다.

원칙:

- 본문은 높은 가독성을 우선한다.
- 제목과 라벨에서만 픽셀 감성을 강화한다.
- 작은 화면에서도 정보 위계가 명확해야 한다.

# 3. 컬러 시스템

## 3.1 메인 컬러 방향

여운의 컬러는 **라벤더 + 핑크 + 하늘색** 중심의 파스텔 팔레트다.  
다만 형광처럼 튀는 색은 피하고, 크림/오프화이트를 넉넉히 사용해 과밀도를 낮춘다.

## 3.2 핵심 팔레트

아래 값은 출발점 기준의 추천값이다. 실제 구현 시 접근성 대비를 확인하며 조정한다.

### Primary

- Lavender 100: `#F3EEFF`
- Lavender 200: `#E6D9FF`
- Lavender 300: `#D3BFFF`
- Lavender 400: `#B995FF`
- Lavender 500: `#9A6DFF`

### Secondary

- Pink 100: `#FFF0F6`
- Pink 200: `#FFD9EA`
- Pink 300: `#FFBEDB`
- Pink 400: `#FF9BC5`
- Pink 500: `#FF73B0`

### Accent

- Sky 100: `#EFF8FF`
- Sky 200: `#D9EEFF`
- Sky 300: `#B9DEFF`
- Sky 400: `#8DC8FF`
- Sky 500: `#5FAEFF`

### Support

- Mint 100: `#EEFDF6`
- Mint 300: `#BDF5DA`
- Butter 100: `#FFF9E8`
- Butter 300: `#FFE9A8`
- Coral 300: `#FFB5AE`

### Neutral

- Cream 50: `#FFFDFC`
- Warm Gray 100: `#F6F2F4`
- Warm Gray 200: `#E8E1E7`
- Warm Gray 400: `#9A8F99`
- Ink 700: `#4F4752`
- Ink 900: `#2F2932`

## 3.3 컬러 사용 규칙

- 배경의 기본값은 화이트가 아니라 **크림 화이트 또는 아주 연한 라벤더**를 사용한다.
- 강한 컬러는 버튼, 태그, 선택 상태, 반응 상태에서만 제한적으로 사용한다.
- 한 화면 안에서 주도권을 가지는 강색은 1~2개만 허용한다.
- 카드가 많을 때는 카드 배경을 흰색 또는 매우 옅은 톤으로 유지해 과밀감을 줄인다.

## 3.4 상태 컬러

### Positive / Active

- Mint 300 또는 Lavender 400

### Warning

- Butter 300

### Danger / 신고 / 삭제

- Coral 300 또는 Pink 400 기반 경고색

### Disabled

- Warm Gray 200 배경 + Warm Gray 400 텍스트

# 4. 타이포그래피 시스템

## 4.1 기본 원칙

여운은 픽셀 감성을 가지고 있지만, 모든 텍스트를 픽셀 폰트로 처리하면 가독성이 나빠진다.  
따라서 **브랜드/라벨/반응명은 픽셀 감성**, **본문/설명/입력은 읽기 쉬운 UI 폰트**로 구분한다.

## 4.2 역할 분리

### A. Display / Branding

용도:

- 로고
- 주요 섹션 제목
- 반응 라벨(슬쩍/콕/꾹/폭닥/젤리)
- 이벤트성 라벨

조건:

- 네모네모한 비트맵/도트 느낌
- 너무 깨지지 않아야 함
- 작은 크기에서도 어느 정도 읽혀야 함

### B. UI / Body

용도:

- 본문
- 댓글
- 입력폼
- 설정
- 설명 문구

조건:

- 모바일에서 높은 가독성
- 지나치게 딱딱하지 않은 산세리프
- 한글 자간이 답답하지 않은 서체

## 4.3 텍스트 스케일

### Display XL

- 용도: 온보딩 메인 헤드라인, 큰 섹션 타이틀
- 권장 크기: 28~32
- weight: bold

### Heading L

- 용도: 화면 제목
- 권장 크기: 22~24
- weight: semibold/bold

### Heading M

- 용도: 카드 제목, 모달 제목
- 권장 크기: 18~20
- weight: semibold

### Body L

- 용도: 본문, 설명
- 권장 크기: 16
- weight: regular/medium

### Body M

- 용도: 일반 보조 문구
- 권장 크기: 14
- weight: regular

### Caption

- 용도: 시간, 참여 수, 보조 라벨
- 권장 크기: 12
- weight: regular/medium

## 4.4 텍스트 컬러 규칙

- 메인 본문: Ink 900
- 보조 본문: Ink 700
- 서브 정보: Warm Gray 400
- 강조 라벨: Lavender 500 또는 Pink 500
- 경고/신고: Coral 계열

## 4.5 타이포 사용 주의사항

- 본문 전체를 픽셀 폰트로 쓰지 않는다.
- 한 화면에 폰트 종류는 2개를 넘기지 않는다.
- 타이틀과 본문 간 위계가 분명해야 한다.
- 과도한 그림자/외곽선 텍스트 효과는 금지한다.

# 5. 아이콘 시스템

## 5.1 방향

여운의 아이콘은 서비스 정체성을 결정하는 핵심 요소다.  
모든 아이콘은 **픽셀 결을 가진 통일된 언어**를 유지해야 한다.

## 5.2 스타일 규칙

- 고정 그리드 기반 제작: 16 / 20 / 24 / 32 px 기준
- 1~2px 외곽선 느낌 유지
- 과한 내부 디테일 금지
- 선 굵기와 모서리 표현 통일
- 그림자는 최소화
- 모든 아이콘이 같은 가족처럼 보여야 함

## 5.3 기본 아이콘 세트

- 홈
- 산책
- 업로드
- 편지함
- 무드카드
- 저장/보관함
- 알림/방울
- 검색
- 설정
- 더보기
- 뒤로가기
- 닫기
- 댓글
- 위치
- 태그
- 신고
- 차단

## 5.4 브랜드 특화 아이콘

- 발바닥
- 젤리
- 리본 카메라
- 편지 봉투
- 스탬프/도장
- 찰나 프레임
- 무드 태그 스티커

## 5.5 금지 규칙

- 하트 중심 아이콘 체계 금지
- 실사 스타일, 글로시 아이콘 금지
- 다른 앱에서 익숙한 아이콘 실루엣을 거의 그대로 복제하지 않기

# 6. 반응 시스템 비주얼

## 6.1 여운 5단계

- 1발: 슬쩍
- 2발: 콕
- 3발: 꾹
- 4발: 폭닥
- 5발: 젤리

## 6.2 비주얼 표현 원칙

여운 반응은 숫자 평점이 아니라 **여운을 남기는 행위**로 느껴져야 한다.

원칙:

- 숫자보다 발바닥 모양이 먼저 보인다.
- 반응을 누를 때 탭감이 느껴져야 한다.
- 5단계로 갈수록 더 말랑하고 만족스러운 피드백을 준다.

## 6.3 상태별 표현

### Default

- 비워진 발바닥 또는 아주 옅은 채움

### Hover / Focus

- 라벤더 또는 핑크 계열 미세 강조

### Selected

- 단계에 따라 발바닥이 순차적으로 채워짐
- 선택한 라벨 텍스트 강조

### 5발(젤리)

- 미세한 팝 애니메이션
- 젤리 색감 강조
- 과도한 파티클은 금지, 작고 짧게

## 6.4 평균 감도 라벨

표기 예시:

- 젤리 · 평균 감도 4.72발
- 폭닥 · 평균 감도 4.18발

원칙:

- 라벨이 먼저, 수치가 뒤에 온다.
- 피드에서는 짧게, 상세에서는 더 정확하게 보여준다.
- 숫자만 크게 부각하지 않는다.

# 7. 레이아웃 시스템

## 7.1 그리드

모바일 우선 설계를 기본으로 한다.

권장:

- 4pt 또는 8pt spacing system
- 좌우 패딩은 16~20 기준
- 카드 내부 패딩은 12~16 기준
- 섹션 간격은 20~32 기준

## 7.2 화면 구조 원칙

- 헤더는 가볍고 명확해야 한다.
- 사진 영역은 크게 유지한다.
- 주요 액션은 엄지 닿는 영역에 위치시킨다.
- 같은 종류의 정보는 같은 위치에 배치한다.

## 7.3 카드 구조 원칙

홈/피드 카드 기본 구조:

1. 작성자 영역
2. 사진 영역
3. 대표 반응 + 평균 감도
4. 액션 영역
5. 짧은 설명

## 7.4 카드 표현 규칙

- 과한 그림자 금지
- 테두리는 얇고 부드럽게
- 카드 안에 카드가 중첩되지 않게 한다.
- 사진 프레임은 지나치게 두껍지 않게 한다.

# 8. 컴포넌트 시스템

## 8.1 버튼

### Primary Button

용도:

- 업로드 완료
- 저장
- 다음 단계

스타일:

- Lavender 400~500 배경
- 흰색 또는 진한 잉크 텍스트
- 너무 둥근 캡슐형보다 적당한 rounded shape
- 크기는 손가락으로 누르기 충분해야 함

### Secondary Button

용도:

- 취소
- 필터
- 경량 액션

스타일:

- Cream 또는 White 배경
- 얇은 Lavender/Warm Gray border

### Ghost Button

용도:

- 카드 내 서브 액션
- 텍스트형 링크 액션

스타일:

- 배경 없음
- hover/press에서만 미세 배경

## 8.2 탭바

하단 탭바는 여운 정체성에 중요하다.

탭:

- 홈
- 산책
- 업로드
- 편지함
- 무드카드

원칙:

- 아이콘 + 짧은 라벨 구성
- 활성 탭은 컬러 강조
- 비활성 탭은 저명도 뉴트럴
- 업로드는 중앙 강조 가능

## 8.3 태그 칩

태그는 말풍선보다 **스티커/라벨** 느낌이 적합하다.

원칙:

- 크기는 작고 읽기 쉬워야 한다.
- 색상은 태그마다 다르게 하되 과하지 않게 한다.
- 선택 상태는 배경+테두리+텍스트로 명확히 구분한다.

## 8.4 입력 필드

입력 UI는 귀엽지만 실사용성이 높아야 한다.

원칙:

- 넉넉한 높이
- 선명한 focus state
- placeholder는 지나치게 연하지 않게
- 픽셀 장식을 과도하게 넣지 않음

## 8.5 모달 / 바텀시트

용도:

- 반응 선택 안내
- 신고 사유 선택
- 업로드 옵션
- 찰나 시간 선택

원칙:

- 모바일에서 바텀시트 우선
- 배경 블러는 약하게
- 닫기/취소가 명확해야 함

## 8.6 프로필 카드(무드카드)

용도:

- 내 프로필 / 타인 프로필 헤더

구성:

- 프로필 이미지
- 닉네임
- 자기소개
- 대표 태그
- 평균 감도
- 팔로우/편지 버튼

# 9. 화면별 디자인 가이드

## 9.1 홈

목표:

- 익숙한 SNS 피드처럼 사용되되, 들어오자마자 여운 세계관이 보인다.

포인트:

- 헤더는 단정하게
- 찰나는 상단 가로 스크롤
- 메인 카드에서 사진이 가장 크게 보여야 함
- 반응 시스템은 카드 하단에서 빠르게 접근 가능해야 함

## 9.2 산책

목표:

- 새로운 사진과 사람을 둘러보는 발견 중심 화면

포인트:

- 그리드 기반
- 태그 탐색이 쉬워야 함
- 너무 정보가 많지 않게
- 인기/최신/태그별 흐름이 명확해야 함

## 9.3 업로드

목표:

- 사진을 올리는 과정이 쉽고 기분 좋아야 한다.

포인트:

- 단계가 많아 보이지 않게 설계
- 사진 선택이 가장 먼저
- 무드 태그와 설명은 부담 없이 작성 가능하게
- 찰나/일반 게시물 전환이 명확해야 함

## 9.4 편지함

목표:

- DM이지만 과하게 무겁거나 위험해 보이지 않아야 한다.

포인트:

- 요청함/진행중 대화 분리
- 프로필 연결이 쉬워야 함
- 안전 기능(차단/신고) 노출이 명확해야 함

## 9.5 무드카드

목표:

- 프로필이 이력서보다 취향 카드처럼 보여야 한다.

포인트:

- 사진 그리드와 무드 정보의 밸런스
- 평균 감도, 대표 무드, 찰나 상태가 자연스럽게 드러나야 함

# 10. 모션 시스템

## 10.1 원칙

여운의 모션은 작고 말랑해야 한다.  
과한 튐, 긴 애니메이션, 화려한 파티클은 금지한다.

## 10.2 사용 예시

- 탭 전환: 짧고 부드러운 fade/slide
- 버튼 탭: 미세한 scale down/up
- 여운 선택: 순차 채움 + 짧은 강조
- 젤리 선택: 짧은 pop
- 찰나 진행 상태: 조용한 progress reduction

## 10.3 지양할 것

- 과한 bounce
- 무거운 parallax
- 1초 이상 길게 지속되는 피드백 애니메이션

# 11. 일러스트 / 캐릭터 가이드

## 11.1 방향

캐릭터는 앱을 대표하는 마스코트 역할을 할 수 있으나, UI보다 앞서면 안 된다.

원칙:

- 2~3등신의 말랑한 실루엣
- 눈/입 단순화
- 장식 1개만 사용
- 화면 장식은 가장자리나 empty state 중심

## 11.2 사용 위치

- 온보딩
- 빈 상태 화면
- 알림이 없을 때
- 업로드 완료 축하

## 11.3 주의

- 피드/상세 화면 본문에서 캐릭터가 사진보다 시선을 빼앗지 않게 한다.

# 12. 접근성 가이드

## 12.1 대비

- 본문 대비는 최소한 충분히 확보한다.
- 파스텔 배경 위 텍스트는 무조건 대비 검토 후 사용한다.

## 12.2 터치 영역

- 모바일 터치 영역은 충분히 확보한다.
- 작은 픽셀 아이콘이라도 실제 hit area는 넓게 잡는다.

## 12.3 텍스트 크기

- 본문 14 미만 남용 금지
- 중요한 정보는 16 이상 우선 사용

## 12.4 의미 전달

- 컬러만으로 상태를 구분하지 않는다.
- 아이콘 + 텍스트를 함께 사용한다.

# 15. AI 디자인/개발 에이전트용 규칙 요약

다음 규칙은 Codex, Cursor, Claude Code, Antigravity 등에서 여운 UI를 생성할 때 반드시 지켜야 한다.

## 15.1 반드시 지킬 것

- mobile-first
- pastel pixel aesthetic
- soft, airy spacing
- photo-first layout
- cat paw reaction system as primary interaction
- no heart-centered interaction model
- no generic startup gradient UI
- no card-on-card clutter
- use playful but readable typography
- consistent pixel icon language

## 15.2 절대 하지 말 것

- 인스타 레이아웃을 거의 그대로 복제
- 틴더식 카드/매칭 구조를 노골적으로 재현
- purple gradient slop
- shadow-heavy dribbblish cards
- childish toy-like stickers everywhere
- body text in hard-to-read pixel font

## 15.3 결과물 체크리스트

- 첫 화면에서 여운 세계관이 보이는가?
- 사진이 장식보다 먼저 보이는가?
- 발바닥 반응이 하트보다 더 자연스럽게 느껴지는가?
- 라벨/태그/알림 카피가 서비스 톤과 맞는가?
- 귀엽지만 앱으로서 충분히 정돈되어 보이는가?
