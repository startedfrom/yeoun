# 조각찾기(piece-find) — 구현 레퍼런스

## 제품 정체성 (위반 금지)

- 패션 포토·해시태그·소프트 파스텔 픽셀·**여운(발바닥)** 반응 세계관 안에서만 말할 것.
- **데이팅 클론 금지**: X/하트/슈퍼라이크/매칭/VS 대형 타이포/중앙 하트/양손 스와이프 메타포.
- 카피는 **「룩·핏·무드·조각 수집」** — 사람 심판·호감 표현 금지.

## 핵심 루프

1. 산책 탭에서 진입 CTA → 조각찾기 세션.
2. **패션 적격** 포스트 **정확히 2장** 동시 노출(좌우 또는 상하, 대칭).
3. 사용자가 **한쪽에 여운** → 선택 확정 → **별도 ‘다음’ 없이 즉시** 다음 쌍으로 교체.
4. **편지(유료)**는 **여운 확정 전에만** 노출·발송 가능. 반응 후에는 비활성 + 이유 안내.
5. 제외 규칙은 **서버 권위**로 강제(클라만 믿지 말 것).

## 제외·필터 (후보 풀)

다음에 해당하는 `postId`는 해당 유저의 조각찾기 후보에서 **제외**:

- 현재 유저 본인 작성.
- 차단 관계(양방향).
- 신고·모더레이션으로 숨김/제한된 포스트.
- 유저가 **어디서든** 이미 여운 반응을 남긴 포스트.
- 유저가 조각찾기에서 **유료 편지를 보낸** 포스트.
- 라이프스타일-only / 풍경-only 등 **패션 비적격**(태그 화이트리스트 + 운영 플래그/모델 게이트 중 팀이 택한 기준을 코드에 반영).

## 페어·다양성

- 동일 unordered pair `(A,B)` == `(B,A)` → **동일 쌍**으로 취급해 재노출 방지.
- 연속 N쌍 내 동일 `authorId` 상한, 세션·일 단위 작가 쿼터.
- 대표 해시태그가 너무 겹치는 쌍은 샘플링에서 페널티.

## 노출만 한 사진(MVP 권장안)

- **스킵 확정** 또는 **반응/편지 확정** 시에만 쌍의 양쪽 `postId`를 “조각찾기 소진” 집합에 넣는다.
- 단순 이탈(abandon)은 개인화 약신호로만 로그하고, **영구 제외는 과도**할 수 있음(팀 정책에 맞게 조정).

## 유료 편지

- 페이월은 **「편지 보내기」 탭 시점**에 명시(몰래 끼워 넣기 금지).
- 일일·세션 상한, 동일 수신자 쿨다운, idempotency·환불/재시도 정책.
- 편지 성공 시 해당 `postId`는 조각찾기 후보에서 영구 제외(요구사항).

## 상태 머신 (요약)

- `loading_pair` → `pair_ready` → (`optional_letter_open`) → `reacted` → `advance` → `pair_ready` …
- `empty_pool` / `error_network` / `payment_cancelled` / `letter_failed` 각각 복구 동선 정의.

## 개인화 시그널(저장)

- `reaction`에 `source=piece_find` 구분.
- 선택된 포스트의 태그·(가능하면) 스타일 메타, 스킵, 세션 길이.

## UI 구현 시

- 모션은 **transform/opacity** 위주; 레이아웃 값 애니메이션 남발 금지.
- impeccable **absolute_bans** 준수(카드 왼쪽 굵은 컬러 스트라이프, 그라데이션 텍스트 등).
